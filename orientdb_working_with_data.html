    OrientDB - Working with the Data

    IB-Ruby translates data received from the TWS in IB::Model objects which are subsequently stored in database classes prefixed by 'ib'(Namespace: IB).

    The date-grid is implemented by database-classes prefixed by 'tg' (Namespace: TG).

    It is advisable to put calculations and further analyses to another namespace.

    HC::Portfolio

    As living example, HC::Portfolio is included, which is initialized through bake:

    module HC
 class Portfolio
  def self.bake  
   gateway= IB::OrientGateway.current                 # OrientDB-Instance
   gateway.get_account_data                           # ask TWS for portfolio-data
   gateway.clients.map do | the_client |              # perform task for all clients
     p = self.new
     p.import_account_data from: the_client           # implies update of p, assigns a rid
     p.import_positions    from: the_client           # assigns portfolio-values to the grid
     HC::MY_USER.create    from: p, to: the_client    # bidirectional link to account
     Date.today.to_tg.assign vertex: p, via: HC::D2F  # bidirectional link to the time-grid
     p.rid                                            # return_value
    end
  end
 end
end

    The last command of the bake-method assigns the portfolio-object via HC::D2F to the time-grid.
    In the process, a bidirectional link TimeGrid -- out HC::D2F -- in -- Portfolio is implemented.

    > "7.12.2020".to_tg.nodes( :out, via: HC::D2F )         => Array of assigned Portfolio's
> HC::Portfolio.last.nodes( :in, via: HC::D2F ).datum  => ["7.12.2020"] # Array of time-grid-objects

    note: Vertex#nodes is a method of ActiveOrient, its documented here

    It is reasonable to implement a shortcut portfolios in models/tg/tag.rb (modelfile of the timegrid).

    module TG
  class Tag
    def portfolios
      nodes( :out, via: HC::D2F )
    end
  end
end

    Then "7.12.2020".to_tg.portfolios yields an array of connected portfolio-records.

    The following database queries are fired:

    ->select COUNT(*) from tg_time_of 
->select  expand (  outE('tg_day_of').in[ value = 7 ]  ) from  ( select  expand (  outE('tg_month_of').in[ value = 12 ]  ) from  ( select from tg_jahr where value = 2020  )   )  ## result: #49:1868 
->select  outE('hc_d2F').in  from #49:1868

    Get Account-Data

    In the IB-Gateway-Section a primitive approach to access Account-Value-Data is presented.

    HC::Portfolio#import_account_data imports IB::AccountValue-Objects as hash into the database.

     HC::Portfolio.values[ :key ][:ALL][:EUR] => value
                                   [:USD] => value
														 [:S]  [:EUR] => Value
														 [:C]  [:EUR] => value

    Thus to access the data from yesterday:

     Date.today.to_tg.prev.portfolios &.values.map{|y| y[:SMA][:ALL][:EUR] }
 => [-9033.37, 12565.67]
 # or
 Date.today.to_tg.prev.portfolios &.values.slice /SM/
 => [{:SMA=>{:ALL=>{:EUR=>-4701.63}, :S=>{:EUR=>-4701.63}}}, (...)

    Database Solution

    The same result is achieved using a simple database query

      select values[ key ] as key from hc_portfolio

    It is composed with OrientSupport::OrientQuery, to be reused in different contexts. In the following example the keys are specified as a regular expression.

    > Q=  OrientSupport::OrientQuery                     => just a shortcut
> p =  HC::Portfolio.last                            => get stored data
> # regular expression:  /Ac/
> fields = p.values.keys.match(/Ac/).compact.string  => ["AccruedCash", "AccruedDividend"] 
# build the projection
> the_projection = fields.map{|y| "values[\"#{y}\"] as #{y} "}.join(", ")
# compose the query  (all entries of HC::Portfolio)
# note: Instead of a database class, a subquery can be provide as well.
> query= q.new from: HC::Portfolio, projection: the_projection
> query.to_s         
=> "select values[\"AccruedCash\"] as AccruedCash , values[\"AccruedDividend\"] as AccruedDividend  from hc_portfolio "
> result =  query.execute
=> [{:AccruedCash=>{:ALL=>{:USD=>-1.41, :BASE=>-1.1629538}, :S=>{:EUR=>-1.16}}, 
     :AccruedDividend=>{:ALL=>{:EUR=>30.43}, :S=>{:EUR=>30.43}}}, 
    {:AccruedCash=>{:ALL=>{:USD=>-1.41, :BASE=>-1.1640194}, :S=>{:EUR=>-1.16}},
