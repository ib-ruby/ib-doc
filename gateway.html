    IB-Gateway

    IB::Gateway is part of ib-extensions. Include 
     
require 'ib-api' 
require 'ib/gateway'
    in your scripts.

    Typical setup

    require 'bundler/setup'
require 'ib-api'
require 'ib-gateway'

TWS= 'tws:7496' # or 'localhost'  

IB::Gateway.new host: TWS , client_id:1134

if ( ib = IB::Gateway.tws ).present?
    (...)
  else
   puts 'No TWS'
  end
    IB::Gateway keeps Account- and Portfolio-Data, Contracts, Orders in an in-line database and provides account-based trading tools.

    Upon initialization, it detects accounts and reads available data from the TWS.

    Secondly, it aims to provide an intuitive interface for order-placement and -management

    Accounts

    In a financial advisor environment, after initialization IB::Gateway the accounts are present through

    > g =  IB::Gateway.current
> g.advisor.to_human
 => "<demo_advisor DF167347>" 
> puts g.clients.to_human
<demo_user DU167348>
<demo_user DU167349>

    In single user mode Advisor and Client are identical:

    > g =  IB::Gateway.current
> g.advisor.to_human
 => "<user U1274612>" 
> puts g.clients.to_human
<user U1274612>

    • IB::Gateway.tws points to the active and instantiated IB::Connection and its methods.

    • IB::Gateway.current links to the instantiated IB::Gateway and its account-database.

    IB::Gateway.clients contain Account-Positions and Portfolio-Values. They are synchronized with the TWS by IB::Gateway.get_account_data {the account}. Visit Account Overview for further details.

    Order Management

    A simplified order management process is defined

    > G =  IB::Gateway.new   # initialize with default parameter
> the_contract =  IB::Symbols::Stocks.msft  # Microsoft
> the_order = Limit.order size: 100, 
                          price: the_contract.market_price - 1 ,
                          contract:  the_contract
> the_order.to_human
 => "<Order: LMT GTC buy 100  @ 201.33 New #/ from >" 

> the_account  =  G.clients.first

> the_account.preview order: the_order, 
                      contract: the_contract
  => {:init_margin=>0.1344327e5, :maint_margin=>0.1168288e5, 
      :equity_with_loan=>0.118936284e7, :commission=>nil, 
      :commission_currency=>"USD", :warning=>""} 

> the_account.place order: the_order

    The general procedure for order-tasks:

    > the_account.{ preview | place | modify | reverse | close  } order: the_order, 
                                                              contract: the_contract
    :: The order-object is updated in the process. It’s OK to use the same order-object for each step. Then the history is kept in OrderState-Objects. But never use the same order object for different users.
