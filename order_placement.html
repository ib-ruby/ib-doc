    Place, Modify and Cancel Orders

    To use the features described, include the ib-extensions Gem in the Gemfile and use 
     
require 'ib-api' 
require 'ib/symbols' 
require 'ib/order-prototypes' 
require 'ib/gateway'
    in your scripts.

    Place by sending a Message

    To place an order, basicly a :PlaceOrder- Message has to be transmitted

    > ib =  IB::Connection.current
> ib.send_message :PlaceOrder, contract: IB::Symbols::Stocks.wfc , 
                   order: Limit.order( action: :buy, size: 100, price: 50  ),
                   local_id: ib.next_local_id

    TWS responses with :OpenOrder and OrderStatus messages.

    The PlaceOrder message is also used to modify orders. A new order requires an incremented :local_id. By transmitting a :PlaceOrder-Message with an unchanged :local_id the order is modified (without warning).

    Place with IB::Connection-methods

    To avoid unintended order-modifications, explicit calls to :place_order and :modify_order are implemented.

    IB-Api provides handy IB::Connection#place_order and IB::Connection#modify_order methods.

    > ib =  IB::Connection.current
> the_order =  Limit.order action: :buy, size: 100, price: 50
> the_contract = IB::Symbols::Stocks.wfc 
  # call _place_order_ and get the current _order_id
> order_id = ib.place_order the_order, the_contract 
	# adjust the order and transmit the modification 
> the_order.price = 51
> the_order.local_id = order_id
> ib.modify_order the_order, the_contract

    The response is stored in IB::Connection.received.

    Preview and Place with IB::Gateway

    IB::Gateway handles orders at the account-level. Thus orders are explicitly directed to a specific account.

    > IB::Account.preview contract: {an IB::Contract} , order: { a IB::Order  } 
	>> returns a forcast for the action

> IB::Account.place contract: {an IB::Contract} , order: { a IB::Order  } 
	>> places the order and returns the `local_id'

    Order-Objects can establish a link to a contract. Then only one argument is required to initiate the process. IB::Account defineds two methods that ease the process:

    > G =  IB::Gateway.current
> the_order =  Limit.order action: :buy, size: 100, price: 45, 
                           contract: IB::Symbols::Stocks.wfc
> G.active_accounts[1].preview order: the_order 
  => {:init_margin=>0.9383838e5, :maint_margin=>0.8545908e5, 
      :equity_with_loan=>0.98950817e6, :commission=>0.1e1,    
      :commission_currency=>"USD", :warning=>""} 
> G.active_accounts[1].place order: the_order   
  => 123   # local_id
    More details on reviewing orders: preview

    If specified as part of an Order (as in the upper example), IB::Contracts are transmitted without any modification . If specified separately, the IB::Contract is proper verified and the price is adjusted to the min_tick requirements. The transmitted IB::Contract is kept in Order.contract and thus available for a later inspection.

    Locate Order

    After placing the Order, OrderState-Messages are received and then linked to the IB::Order-object. This is stored in IB::Account.orders and can easily accessed via IB::Account.locate_order.

    > IB::Account.locate_order local_id:, perm_id:, con_id:, status: /ubmitted/ 
 >> Gets the first order matching the criteria  

    > G.active_accounts[1].orders.to_human
=> ["<Order: LMT GTC buy 100 45.0 New #1/ from 3000/DU167349>",
  "<Order: LMT GTC buy 100.0 45.0 Submitted #1/693213230 from 3000/DU167349>"]
> G.active_accounts[1].locate_order( local_id: {local_id}  ) &.limit_price
  => 0.45e2
> G.active_accounts[1].locate_order( con_id: {contract.con_id}  ) &.limit_price
  => 0.45e2

    Modify

    IB::Account.place updates the local_id attribute of the transmitted IB::Order-object.
    To modify the order, just change the desired attributes and call IB::Account.modify

    > the_order.total_quantity = 600
  => 600 
> G.active_accounts[1].modify_order order: the_order
  => 1 
> G.active_accounts[1].orders.to_human
  => ["<Order: LMT GTC buy 600 45.0 New #1/ from 3000/DU167349>", 
      "<Order: LMT GTC buy 600.0 45.0 Submitted #1/693213230 from 3000/DU167349>"]

    Impossible Modification

    If the TWS cannot process the desired modification, an :Alert-Message is fired

    Notice: According to the Interactive Brokers Documentation, “it is not generally recommended to try to change order fields aside from order price, size, and tif (for DAY -> IOC modifications). To change other parameters, it might be preferable to instead cancel the open order, and create a new one.”

    > the_order.action =  :sell
> G.active_accounts[1].modify_order order: the_order 
> Got message 4 (IB::Messages::Incoming::Alert)
 ----------------105-----
 TWS Error 105: Order being modified does not match original order
> G.active_accounts[1].orders.to_human
=> ["<Order: LMT GTC sell 600 45.0 New #1/ from 3000/DU167349>", 
	 "<Order: LMT GTC buy 600.0 45.0 Rejected #1/693213230 from 3000/DU167349>"]

    The Account.orders-Array is updated, the last status (Rejected) is noticed, as the still active Order (buy).

    Canceling Orders

    It is sufficient to transmit a :CancelOrder-Message with IB::Order.local_id as parameter.
    TWS responses with an OrderStatus combined with an :Alert-Message

    The Gateway-Application updates the Account.orders-Array accordingly

    > G.cancel_order the_order.local_id
  Got message 3 (IB::Messages::Incoming::OrderStatus)
  Got message 4 (IB::Messages::Incoming::Alert)
  ---------------202-----
							
> G.active_accounts[1].orders.to_human
=> ["<Order: LMT GTC sell 600 45.0 New #1/ from 3000/DU167349>", 
    "<Order: LMT GTC buy 600.0 45.0 Cancelled #1/693213230 from 3000/DU167349>"]

    Now the order-record informs about the successfully canceled order.

    The OrderState-History

    The Gateway Application does some bookkeeping of :OrderState-Messages.

    note: for details look into orderhandling.rb –> IB::Alert#AddOrderstateAlert

    The history of the order-process is stored in Account.orders.order_states

    puts G.active_accounts[1].orders.order_states.to_human.join("\n")
  <OrderState: New #/ from  filled 0/0 at 0/0>
  <OrderState: Submitted #1/693213230 from 3000 filled 0.0/600.0 at 0.0/0.0 why_held >
  <OrderState: Rejected #1/693213230 from  warning 105: Order being modified does not match original order>
  <OrderState: Cancelled #1/693213230 from 3000 filled 0.0/600.0 at 0.0/0.0 why_held >

    The Account.orders-Array is fragile. Once an order is executed or canceled, its contents are outdated. Gateway#update_orders synchronizes the Account.orders-array

    > G.update_orders
> Got message 53 (IB::Messages::Incoming::OpenOrderEnd)
> puts G.active_accounts[1].orders.order_states.to_human.join("\n")
  => nil
